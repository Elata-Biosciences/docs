<Note>
  If you are new to the Elata SDK, start here. This CLI gives you a working app you can run locally before you integrate packages into an existing product.
</Note>

## What this package does

`create-elata-demo` scaffolds demo apps from published Elata templates. It is the fastest way to get a clean reference project running.

You should use it when you want to:

- get to a working Elata app quickly
- start from a known-good baseline
- evaluate the SDK before doing a manual integration
- compare your existing app against a working template when debugging

## Before you begin

- Node.js `18` or higher
- A browser-based project target
- `npm`, `pnpm`, or `npx`

<Info>
  If you run the CLI interactively without `--template`, it prompts you to choose a template. In non-interactive runs, it defaults to `rppg-web-demo`.
</Info>

## Choose a template

Use the template that matches the sensor workflow you want to test first.

| Template | Alias | Best for |
| --- | --- | --- |
| `rppg-web-demo` | `rppg` | Camera-based heart rate in the browser |
| `eeg-web-demo` | `eeg` | EEG signal processing in the browser |
| `eeg-web-ble-demo` | `eeg-ble` | Live Muse-compatible EEG streaming over Web Bluetooth |

<Tabs>
  <Tab title="rPPG">
    Best when you want the fastest browser demo with camera-based biometrics.

    ```bash
    npm create @elata-biosciences/elata-demo my-app
    ```

    Continue with [/sdk/rppg-web/getting-started](/sdk/rppg-web/getting-started) once the app is running.
  </Tab>
  <Tab title="EEG">
    Best when you want browser EEG processing without device transport yet.

    ```bash
    npm create @elata-biosciences/elata-demo my-app -- --template eeg-web-demo
    ```

    Continue with [/sdk/eeg-web/getting-started](/sdk/eeg-web/getting-started) once the app is running.
  </Tab>
  <Tab title="EEG BLE">
    Best when you want live browser streaming from a Muse-compatible headset in Chrome or Edge.

    ```bash
    npm create @elata-biosciences/elata-demo my-app -- --template eeg-web-ble-demo
    ```

    Continue with [/sdk/eeg-web-ble/getting-started](/sdk/eeg-web-ble/getting-started) once the app is running.
  </Tab>
</Tabs>

<Tip>
  You can also use the short aliases `rppg`, `eeg`, and `eeg-ble` with `--template`.
</Tip>

## Create a new demo app

<Steps>
  <Step title="Scaffold the project">
    <Tabs>
      <Tab title="npm">
        ```bash
        npm create @elata-biosciences/elata-demo my-app
        ```
      </Tab>
      <Tab title="pnpm">
        ```bash
        pnpm dlx @elata-biosciences/create-elata-demo my-app
        ```
      </Tab>
      <Tab title="npx">
        ```bash
        npx @elata-biosciences/create-elata-demo my-app
        ```
      </Tab>
    </Tabs>
  </Step>
  <Step title="Pick a template if you want something other than the default">
    ```bash
    npm create @elata-biosciences/elata-demo my-app -- --template eeg
    ```

    To see every available template:

    <Tabs>
      <Tab title="pnpm">
        ```bash
        pnpm dlx @elata-biosciences/create-elata-demo -- --list-templates
        ```
      </Tab>
      <Tab title="npx">
        ```bash
        npx @elata-biosciences/create-elata-demo -- --list-templates
        ```
      </Tab>
    </Tabs>
  </Step>
  <Step title="Install dependencies">
    ```bash
    cd my-app
    pnpm install
    ```
  </Step>
  <Step title="Start the app locally">
    ```bash
    pnpm run dev
    ```

    Open the local URL from your terminal and confirm the template loads correctly.
  </Step>
</Steps>

## What you get

Every generated app includes:

- a minimal Vite \+ React app shell
- Elata packages pinned to a compatible version set
- a template-specific `README.md`
- a buildable reference project you can compare against later

## Working inside a pnpm workspace

<Warning>
  If you scaffold inside another `pnpm` workspace and the new app is not included in that workspace's globs, install and run it from the parent directory with `--ignore-workspace`.
</Warning>

```bash
pnpm --dir my-app --ignore-workspace install
pnpm --dir my-app --ignore-workspace run dev
```

If that does not fit your setup, run `npm install` and `npm run dev` inside the generated app instead.

## Maintainers working in the SDK repo

<Note>
  This section is only relevant if you are working inside the Elata SDK monorepo.
</Note>

Run the package tests:

```bash
pnpm --dir packages/create-elata-demo test
./run.sh test create-elata-demo
```

The second command also smoke-tests each template by scaffolding it, installing dependencies, and running a build.
