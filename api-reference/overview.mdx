# API Reference Overview

This tab documents the only intended public API surface for Elata apps: read-only ownership verification, saved gate evaluation, premium-content ownership, and a small app-scoped discovery layer.

## Base URL

```bash
https://app.elata.bio/api/v1
```

<Tip>
These routes are public. They do not require SIWE, session cookies, bearer tokens, or API keys.
</Tip>

## Stable Surface

### Primitive checks

- `GET /users/{address}/elta`
- `GET /users/{address}/veelta`

### App-scoped verification

- `GET /apps/{tokenAddress}/users/{userAddress}/token`
- `GET /apps/{tokenAddress}/users/{userAddress}/gate-access`
- `GET /apps/{tokenAddress}/users/{userAddress}/premium-content`
- `GET /apps/{tokenAddress}/users/{userAddress}/premium-content/{contentId}`

### Discovery

- `GET /apps/{tokenAddress}/market-summary`
- `GET /apps/{tokenAddress}/premium-content/catalog`
- `GET /apps/{tokenAddress}/premium-content/activity`

## What This API Is For

Use these routes when your app needs to:

- verify whether a wallet currently holds an app token
- evaluate the app's saved gate thresholds against live balances
- check whether a wallet owns specific premium content
- fetch app-scoped catalog or recent-sale data for discovery and bot flows

## What Is Not Public

This API intentionally does **not** expose:

- broad wallet-wide scans
- generic admin or moderation data
- release-management actions
- storefront-management endpoints
- unrelated contract reference material

## Rate Limits

Rate limits are segmented by route family:

| Family | Limit |
| --- | --- |
| Primitive | `120` requests / minute |
| Verification | `90` requests / minute |
| Premium ownership | `60` requests / minute |
| Discovery | `45` requests / minute |

If a limit is exceeded, the API returns `429` and a `Retry-After` header.

## Caching

Elata applies both HTTP caching and short server-side caching:

| Family | Typical cache behavior |
| --- | --- |
| Primitive | `s-maxage=15`, `stale-while-revalidate=30` |
| Verification | `s-maxage=5`, `stale-while-revalidate=5` |
| Premium ownership | `s-maxage=5`, `stale-while-revalidate=5` |
| Discovery | `s-maxage=60`, `stale-while-revalidate=120` |

Append `?fresh=1` to bypass the server-side cache on the next read after a just-confirmed purchase or state change.

## Parameter Notes

- `tokenAddress` and `userAddress` must be valid `0x` Ethereum addresses.
- `contentId` is the store-scoped premium content ID, not the ERC-721 `tokenId`.
- Premium-content routes are only available for apps that have indexed premium-content surfaces configured.

## Network

Elata is currently live on **Ethereum Sepolia** for this public API surface. Base is the intended launch network later, but the current documented public endpoints should describe the running Sepolia-backed environment today.
